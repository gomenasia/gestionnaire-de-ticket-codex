01 Importance du logiciel

Le logiciel est devenu central dans la société et l'économie — "software is eating the world" (Marc Andreessen, 2011). Les systèmes sont de plus en plus massifs et critiques.

📐
Complexité croissante

Des millions à milliards de lignes de code dans les systèmes modernes (Linux, Android, Boeing…).

🏥
Systèmes critiques

Santé, finance, transport : une défaillance logicielle peut avoir des conséquences graves et coûteuses.

🔧
Nécessité du GL

Le génie logiciel fournit méthodes et outils pour maîtriser cette complexité.

02 Qu'est-ce que le génie logiciel ?

Ensemble de méthodes, outils et principes pour produire des logiciels de qualité, dans les coûts et délais impartis.

Fiables
Maintenables

Compétences visées

  • Utiliser l'IA (Copilot, ChatGPT) de manière critique et éclairée

04 Concepts fondamentaux de la POO

Objets & Classes

📦
Objet

Entité avec un état (attributs) et des opérations (méthodes). Combine mémoire et comportements.

🏗️
Classe

Modèle décrivant un type d'objet. Un objet est une instance d'une classe.

📊
UML

Diagrammes de classes et d'objets pour modéliser, documenter et communiquer l'architecture.

05 Modularité, encapsulation & API

Encapsulation

Cacher les détails internes d'un objet. Seule l'interface publique est exposée → facilite la maintenance.

Modularité

Décomposer un système complexe en modules indépendants et réutilisables.

API

Interface publique d'un composant — ce que les autres modules peuvent utiliser sans connaître l'implémentation.

Bug de l'an 2000 (Y2K) : une mauvaise encapsulation de la représentation des dates a coûté des centaines de milliards de dollars de corrections dans le monde entier.

06 Relations entre objets

Association

Relie des classes entre elles. Exprime une dépendance sémantique.

Cardinalités : 0..1, 1..*, *…

Ex : un Client possède un ou plusieurs Comptes.

Composition

Relation forte "partie-de". Les parties n'existent pas sans le tout.

Ex : une Voiture est composée de Roues — détruire la voiture détruit les roues.

Copie & Aliasing

Trois niveaux : affectation (alias), copie simple (shallow), copie profonde (deep).

L'aliasing non intentionnel est une source fréquente de bugs.

07 Héritage & classification

Une sous-classe hérite des attributs et méthodes d'une superclasse, et peut les spécialiser.

🌲
Classification (est-un)

Un Chien est un Animal. L'héritage modélise une hiérarchie de types.

♻️
Réutilisation

Factoriser le code commun dans la superclasse pour éviter la duplication.

📜
LSP

Principe de Liskov : une sous-classe doit pouvoir remplacer sa superclasse sans casser le programme.

Attention : héritage ≠ sous-typage. L'exemple classique : Carré extends Rectangle viole le LSP si setLargeur() et setHauteur() existent indépendamment.
Outils Java

Classes abstraites (abstract class), interfaces (interface), et mot-clé extends / implements.

08 Polymorphisme & liaison dynamique

Une même référence peut désigner des objets de types différents. Le comportement réel est choisi à l'exécution (late binding).

🔌
Polymorphisme d'inclusion

Via l'héritage : IForme f = new Cercle(…) → f.enSVG() appelle la méthode de Cercle.

🔗
Liaison dynamique

La JVM détermine quelle méthode appeler en fonction du type réel de l'objet, pas du type déclaré.

🚀
Extensibilité

Ajouter de nouvelles formes sans modifier le code existant — respecte le principe Open/Closed.

09–10 Héritage multiple & typage

Héritage multiple

Python / C++ / Eiffel

Supportent l'héritage multiple direct. Puissant mais source d'ambiguïtés (problème du diamant).

Java

Héritage simple uniquement. L'héritage multiple est simulé via les interfaces (depuis Java 8 : méthodes default).

Ambiguïté

Si deux parents définissent la même méthode, lequel choisir ? Chaque langage a sa propre règle de résolution.

Typage

🔒
Statique (Java, C++)

Types vérifiés à la compilation. Erreurs détectées tôt, meilleures performances, IDE plus efficace.

🌊
Dynamique (Python, JS)

Types déterminés à l'exécution. Plus flexible, mais erreurs découvertes plus tard.

11 Messages clés

①

Le génie logiciel sert avant tout à gérer la complexité — pas seulement à écrire du code.

②

L'architecture et la collaboration sont aussi importantes que les compétences de codage.

③

L'IA (Copilot, ChatGPT) est un outil utile mais dangereux sans compréhension — comprendre prime sur générer.

④

Un bon logiciel est testable, documenté et maintenable — pas seulement fonctionnel.

👉 Ce cours introduit les bases du génie logiciel et de l'OO pour apprendre à concevoir, structurer, tester et maintenir des logiciels complexes en équipe.
